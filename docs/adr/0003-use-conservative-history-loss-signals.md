# Use conservative history-loss signals

orca-viz reports history loss only from structural evidence that is difficult to produce accidentally, and exposes the affected history surface rather than a generic reset boolean. Message history is signalled by a gap in its persistent sequence, while task graph history is signalled only when every graph-owned collection is empty and retained messages still contain at least one readable task reference; looser orphan-count or ratio heuristics were rejected because ordinary retained history can contain orphaned references indefinitely. The UI describes what is missing and says that the shape *matches* an Orca reset instead of claiming to know which command or actor caused it, accepting false negatives once any new graph row exists in exchange for avoiding persistent false-positive warnings.
